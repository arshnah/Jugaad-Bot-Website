ALPHA — Testers Wanted

Jugaad is in active development. Commands change often, and some will break — that's expected at this stage. The whole point of the alpha is to find those breakages before a stable release. If you run the tests below and report what goes wrong, you're directly moving the bot toward being stable and trustworthy. Thank you for helping.

How to report a bug

If any test below fails or behaves differently than the expected result, message @vasucoded on Discord directly.

Include three things so the bug can actually be fixed:

1. What you ran — the exact command and options (e.g. /coinflip amount:50 side:heads).
2. What you expected — what the checklist said should happen.
3. What actually happened — the real result, including any error text like "The application did not respond." A screenshot helps.

⚠️ Known Issue — don't bother testing or reporting these

11 of the new image/meme commands are currently broken because a required package (@napi-rs/canvas) is listed as a dependency but was never installed. If you run any of these, expect an error or no response at all — that's this known issue, not a new bug:

  • /achievement, /ascii, /caption, /deepfry, /glitch, /jail, /pfp-card, /pixelate, /ship-card, /wanted, /wasted

It's worse than just those 11, though: command loading has no error handling, so if the bot process restarts while this is unfixed, it can crash on startup before it finishes loading commands — taking every command offline, not just the image ones. If you find that nothing responds at all, this is almost certainly why — no need to file a separate report for that either, just let @vasucoded know it's still unfixed.

This checklist covers only the commands that are live in the current build. Planned commands that aren't implemented yet are left out on purpose — there's nothing to test there. Run each one in a server where Jugaad is installed. Where an Edge case is listed, please test it too — those are the situations most likely to expose a bug.

Economy & Jobs

/balance

Action /balance

Expected An embed shows your current Rokda balance. The very first time you run it, your wallet is created with a starting balance instead of an error.

Action /balance user:@anotherperson

Expected Shows that person's balance instead of yours.

/daily

Action /daily

Expected Your balance goes up by somewhere between 150 and 400 Rokda, and an embed confirms the amount and your current daily streak.

Edge case /daily again right away

Expected It refuses and tells you how long until you can claim again. Your balance does not change — you cannot claim twice in one day.

/weekly

Action /weekly

Expected Your balance goes up by 1000–2000 Rokda once, and a 7-day cooldown starts.

Edge case /weekly again right away

Expected Cooldown message with time remaining. No second payout.

/job list

Action /job list

Expected Lists every available job, each with its payout range and cooldown.

/job apply

Action /job apply name: then pick a job from the dropdown

Expected You're assigned that job and get a confirmation. The job comes from a dropdown — you can't type a job name that doesn't exist.

/job stats

Action /job stats

Expected Shows your current job, your XP, and your payout bonus tier. If you haven't applied for a job yet, it tells you to apply first rather than erroring.

/work

Action /work

Expected You earn Rokda for your current job and your balance rises. If you have no job, it prompts you to apply for one first.

Edge case /work again right away

Expected A per-job cooldown message. No extra pay until the cooldown is up.

Gambling

/coinflip

Action /coinflip amount:50 side:heads

Expected Your balance drops by 50 if you lose, or you net +50 (the bet is doubled) if you win. A result embed appears either way — no "application did not respond" error.

Edge case /coinflip amount:999999 side:heads (more than you own)

Expected Rejected for not having enough Rokda; your balance is unchanged. Amounts below 10 or above 250,000 are blocked by Discord before the command even runs.

/slots

Action /slots amount:50

Expected Three reels are shown, your bet is deducted, and you're paid out when symbols match. The embed shows the reels and whether you won or lost.

/blackjack

Action /blackjack amount:100, then use the Hit / Stand buttons

Expected You're dealt a hand against the dealer with working Hit and Stand buttons. The hand resolves to a win, loss, or push and your balance updates. A natural blackjack pays 3:2.

Edge case Start a hand, then run /blackjack amount:100 again before finishing the first

Expected The second attempt is blocked — only one hand at a time. Separately, if you start a hand and just wait, it auto-resolves after 60 seconds instead of getting stuck.

/dice

Action /dice amount:50 mode:higher number:10

Expected A hidden 20-sided die is rolled. You win even money if it lands higher than 10, otherwise you lose. The embed shows the roll.

Action /dice amount:50 mode:exact number:7

Expected You only win if the roll is exactly 7, but the payout multiplier is much bigger than a higher/lower guess.

Edge case /dice amount:50 mode:higher number:25 (outside 1–20)

Expected Discord rejects the number before the command runs — it must be between 1 and 20.

/roulette

Action /roulette amount:50 bet_type:red

Expected The wheel spins and the embed shows which pocket it landed on. A red bet pays 1:1 if it hits.

Action /roulette amount:50 bet_type:number value:17

Expected A straight-up bet on 17 pays 35:1 if the wheel lands on 17.

Edge case /roulette amount:50 bet_type:number with no value

Expected It tells you a number is required and the bet is not placed — your balance stays the same.

/crash

Action /crash amount:50, then press Cash Out

Expected A multiplier climbs from 1.00x. Pressing Cash Out pays your bet times the multiplier at that instant. The embed shows where you cashed out.

Edge case /crash amount:50 and never press Cash Out

Expected When the multiplier crashes you lose your bet — no payout, and the message resolves cleanly rather than hanging.

/lottery buy

Action /lottery buy tickets:1

Expected The ticket cost is deducted, a confirmation shows how many tickets you now hold and the current pool size. The draw happens weekly.

Edge case /lottery buy tickets:99999 (beyond your balance or the per-purchase cap)

Expected Rejected — either the ticket count is capped by Discord or you're told you can't afford it. Your balance is unchanged.

/race

Action /race bet amount:50 horse: then pick a horse

Expected Your bet is placed on that horse and a betting round opens for the server.

Action /race start

Expected Betting closes, the race animates to a finish, and backers of the winning horse are paid.

Edge case /race start with no bets placed, or place a bet and never start

Expected Starting with no open race tells you there's nothing to start. If a round is left unstarted, it auto-cancels after a short wait and refunds everyone who bet.

/elimination

Action /elimination start, then react to join

Expected A join message appears; reacting enters you and deducts the entry fee. After the join window, entrants are removed one at a time until one player remains and takes the pot.

Edge case Start an event where fewer than 2 people join

Expected The event cancels and everyone who joined is refunded their entry fee.

Action /elimination config enabled:true

Expected Only works if you have the Manage Server permission; it turns the short-timeout penalty for eliminated players on or off. Members without that permission get a permission error instead.

Moderation

These commands only appear in Discord's slash-command list for members who hold the matching permission (e.g. Ban Members for /ban). If you can't see a command at all, that's the permission gate working correctly — not a bug. Ask a server admin to grant you a staff role with the needed permissions before testing the rest of this section. Test these on a throwaway account or a member who's expecting it, not a real regular.

/warn

Action /warn user:@testaccount reason:testing

Expected A private reply confirms the warning number, and an entry appears in the mod-log channel (if one is configured).

Edge case Run /warn on the same user 3 times total

Expected On the 3rd warning, the bot also applies a 10-minute timeout automatically and says so in the reply — as long as your role is above the target's.

Edge case /warn user:@yourself reason:testing

Expected Rejected — you can't warn yourself.

/warnings

Action /warnings user:@testaccount (after giving them at least one warning)

Expected Lists each warning with its reason, who issued it, and when — most recent 15 if there are more.

Edge case /warnings user:@someoneneverwarned

Expected Says the user has no warnings, instead of an empty or broken embed.

/kick

Action /kick user:@testaccount reason:testing

Expected The member is removed from the server, you get a private confirmation, and it's logged to the mod-log channel.

Edge case /kick user:@someonewithahigherrole

Expected Rejected with a message about role hierarchy — nobody gets kicked.

/ban and /unban

Action /ban user:@testaccount reason:testing

Expected The member is banned, you get a private confirmation, and it's logged.

Action /unban user-id:<their ID> reason:testing

Expected The ban is lifted and confirmed. You'll need the numeric user ID, not their username.

Edge case /unban user-id:<an ID that was never banned>

Expected Says that user isn't banned — no error, no false success.

/timeout

Action /timeout user:@testaccount duration:5 reason:testing

Expected The member can't send messages or react for 5 minutes; the reply confirms the duration and shows when it lifts.

Edge case /timeout user:@yourself duration:5

Expected Rejected — you can't time yourself out.

/purge

Action Post a few throwaway messages, then /purge amount:5

Expected The 5 most recent messages in the channel disappear and the bot confirms how many were deleted.

Edge case /purge amount:150

Expected Discord blocks the option before the command runs — 100 is the max.

/slowmode

Action /slowmode seconds:10, then try sending two messages quickly

Expected Discord itself enforces a 10-second gap between your messages in that channel. The bot's confirmation posts publicly (not just to you), since it's a visible channel setting.

Action /slowmode seconds:0

Expected Slowmode is disabled and the bot confirms it.

/lock and /unlock

Action /lock reason:testing in a test channel

Expected @everyone can no longer send messages in that channel; the bot posts a public embed confirming the lock.

Action /unlock reason:testing in the same channel

Expected @everyone can send messages again; a public embed confirms the unlock.

/lockdown and /unlockdown

Action /lockdown reason:testing, then press Confirm

Expected A warning prompt appears first, listing how many channels will be affected. After confirming, every text/announcement/forum channel locks and the bot reports how many succeeded.

Action /unlockdown reason:testing, then press Confirm

Expected Same flow in reverse — every channel unlocks.

Edge case Run /lockdown again, but press Cancel (or just let the prompt sit for 30 seconds)

Expected Cancel: nothing is locked, and the bot says so. Timeout: same outcome after 30 seconds pass with no click.

/nickname reset

Action Set a nickname on a test account, then /nickname reset user:@testaccount

Expected The nickname clears back to their username, and the bot confirms it privately.

Edge case /nickname reset user:@someonewithnonickname

Expected Says that member doesn't have a nickname set — nothing changes.

/modlog set

Action /modlog set channel:#mod-log

Expected Confirms the channel is set. Running any moderation command afterward (warn, kick, ban, etc.) should now post a log entry there.

Edge case /modlog set channel: pointed at a channel the bot can't post in

Expected Rejected with a message saying the bot can't send messages there — the setting isn't saved.
